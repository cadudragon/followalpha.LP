## Hard rules

1. **Dependency direction is law.** `Domain` references nothing. `Application` references only `Domain`. `Infrastructure` references `Application` + `Domain`. Hosts (`Api`, `Cli`, `DataSync`) reference `Application`, and `Infrastructure` only in the DI composition root. Architecture tests enforce this; if they fail, fix your code, never the tests.
2. **Domain stays pure.** No I/O, no `DateTime.Now`/`DateTime.UtcNow` (use `IClock` from Application via parameters), no `Random`, no logging, no async. Deterministic functions and immutable types only.
3. **Golden tests are immutable.** The expected values in `tests/FollowAlpha.LP.Domain.Tests/Golden/` are generated by running the Elsts Python reference (`uniswap-v3-liquidity-math`) as an **oracle** via the script in `tools/oracle/`. Never alter expected values or tolerances to make a test pass ŌĆö the C# kernel converges to the oracle, never the reverse. **Product code never shells out to Python**; the Python app is oracle + manual exploration only (principal's decision, 2026-06-12).
4. **This tool never executes transactions.** Read-only on-chain access. No private keys, no signing code, no wallet-connect server side. If a task seems to require signing, stop ŌĆö you misread it.
5. **The decision log is append-only.** Verdicts and channel events are inserts; never update or delete. Schema changes require an explicit migration with a note in the PR description.

   **Naming convention for the verdict (decided 2026-06-14):** C# enum member `Verdict.DoNotOpen`; serialized/wire value and DB constant `DONT_OPEN`; UI label `DON'T OPEN`. Same three-layer pattern for any term whose UI label has punctuation. Domain enum names are the source; serialization maps to the wire constant; the apostrophe lives only in the presentation label.
6. **Collected on-chain data are immutable facts.** Snapshot and event tables are append-only; re-ingestion must be idempotent (natural keys / upserts that cannot change history).
7. **No secrets in the repo.** The Graph gateway API key, RPC URLs with keys: environment variables / user-secrets only. `.env*` is gitignored.
8. **Single user today, SaaS-shaped tomorrow.** Every persisted aggregate carries a `TenantId` (constant default tenant for now). No other multi-tenant infrastructure ŌĆö and no design that would prevent it.
9. **Quality gates**: `nullable` enabled, warnings as errors, all tests green, architecture tests green ŌĆö before every commit. Run: `dotnet build && dotnet test`.
10. **Don't gold-plate.** Implement the current phase's definition of done; resist adding speculative abstractions not named in the contracts. Extensibility lives exactly where `docs/ARCHITECTURE.md` puts the ports, nowhere else.

## Conventions

- .NET 10, C# latest, file-scoped namespaces, `var` where type is obvious.
- Value objects as `readonly record struct` / `record` as appropriate; entities with private setters.
- Tests: xUnit + FluentAssertions; mocks only at ports (NSubstitute or Moq, pick one and stay consistent).
- Money/price math: `decimal` for human-scale values; `System.Numerics.BigInteger` for raw on-chain integers (liquidity L, Q64.96 sqrtPriceX96). Conversions live in one place (`Domain/Primitives`).
- Commit messages: imperative, scoped (`domain: add tick<->sqrtPrice conversion with golden tests`).
- Frontend (`frontend/`) is a separate Next.js app; it talks to the API via the generated OpenAPI client only. No business logic in the frontend ŌĆö verdicts come from the API.

## When blocked

If a contract is ambiguous on a **product/research** question (e.g., a formula's benchmark, an intent rule), stop and ask the principal ŌĆö do not decide. If it's a **mechanical** question (naming, folder, library minor version), resolve it in the spirit of the contracts and note the decision in the PR description.
